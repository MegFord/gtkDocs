Gtk.UIManager::get_widget

function vfunc_get_widget(path:String):Gtk.Widget {
}

Looks up a widget by following a path. The path consists of the names specified in the XML description of the UI. separated by '/'. Elements which don't have a name or action attribute in the XML (e.g. &lt;popup&gt;) can be addressed by their XML element name (e.g. "popup"). The root element ("/ui") can be omitted in the path.

Note that the widget found by following a path that ends in a &lt;menu&gt; element is the menuitem to which the menu is attached, not the menu itmanager.

Also note that the widgets constructed by a ui manager are not tied to the lifecycle of the ui manager. If you add the widgets returned by this function to some container or explicitly ref them, they will survive the destruction of the ui manager.

Since 2.4

path

a path

Returns

the widget found by following the path, or null if no widget was found.
