Gtk.CellRenderer::editing-started

function callback(cell_renderer, editable:Gtk.CellEditable, path:String, ):void;

This signal gets emitted when a cell starts to be edited. The intended use of this signal is to do special setup on editable, e.g. adding a Gtk.EntryCompletion or setting up additional columns in a Gtk.ComboBox.

Note that GTK+ doesn't guarantee that cell renderers will continue to use the same kind of widget for editing in future releases, therefore you should check the type of editable before doing any specific setup, as in the following example: |[ static void text_editing_started (GtkCellRenderer *cell, GtkCellEditable *editable, const gchar *path, gpointer data) { if (GTK_IS_ENTRY (editable)) { GtkEntry *entry = GTK_ENTRY (editable); /&ast; ... create a GtkEntryCompletion &ast;/ gtk_entry_set_completion (entry, completion); } } ]|

Since 2.6

cell_renderer

instance of Gtk.CellRenderer that is emitting the signal

editable

the Gtk.CellEditable

path

the path identifying the edited cell
