Gtk.Entry::populate-popup

function callback(entry, popup:Gtk.Widget, ):void;

The ::populate-popup signal gets emitted before showing the context menu of the entry.

If you need to add items to the context menu, connect to this signal and append your items to the @widget, which will be a Gtk.Menu in this case.

If Gtk.populate-all is true, this signal will also be emitted to populate touch popups. In this case, @widget will be a different container, e.g. a Gtk.Toolbar. The signal handler should not make assumptions about the type of @widget.

entry

instance of Gtk.Entry that is emitting the signal

popup

the container that is being populated
