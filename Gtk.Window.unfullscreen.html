Gtk.Window.prototype.unfullscreen

function unfullscreen():void {
    // Gjs wrapper for gtk_window_unfullscreen()
}

Asks to toggle off the fullscreen state for window. Note that you shouldn't assume the window is definitely not full screen afterward, because other entities (e.g. the user or <link linkend="gtk-X11-arch">window manager</link>) could fullscreen it again, and not all window managers honor requests to unfullscreen windows. But normally the window will end up restored to its normal state. Just don't write code that crashes if not.

You can track the fullscreen state via the "window-state-event" signal on Gtk.Widget.

Since 2.2
