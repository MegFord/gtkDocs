Gtk.PrintOperation::preview

function callback(print_operation, preview:Gtk.PrintOperationPreview, context:Gtk.PrintContext, parent:Gtk.Window, ):Boolean;

Gets emitted when a preview is requested from the native dialog.

The default handler for this signal uses an external viewer application to preview.

To implement a custom print preview, an application must return true from its handler for this signal. In order to use the provided context for the preview implementation, it must be given a suitable cairo context with Gtk.PrintContext.prototype.set_cairo_context.

The custom preview implementation can use Gtk.is_selected and Gtk.render_page to find pages which are selected for print and render them. The preview must be finished by calling Gtk.end_preview (typically in response to the user clicking a close button).

Since 2.10

print_operation

instance of Gtk.PrintOperation that is emitting the signal

preview

the Gtk.PrintOperationPreview for the current operation

context

the Gtk.PrintContext that will be used

parent

the Gtk.Window to use as window parent, or null

Returns

true if the listener wants to take over control of the preview
