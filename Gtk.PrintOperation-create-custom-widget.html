Gtk.PrintOperation::create-custom-widget

function callback(print_operation, ):GObject.Object;

Emitted when displaying the print dialog. If you return a widget in a handler for this signal it will be added to a custom tab in the print dialog. You typically return a container widget with multiple widgets in it.

The print dialog owns the returned widget, and its lifetime is not controlled by the application. However, the widget is guaranteed to stay around until the Gtk.custom-widget-apply signal is emitted on the operation. Then you can read out any information you need from the widgets.

Since 2.10

print_operation

instance of Gtk.PrintOperation that is emitting the signal
