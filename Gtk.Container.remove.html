Gtk.Container.prototype.remove

function remove(widget:Gtk.Widget):void {
    // Gjs wrapper for gtk_container_remove()
}

Removes widget from container. widget must be inside container. Note that container will own a reference to widget, and that this may be the last reference held; so removing a widget from its container can destroy that widget. If you want to use widget again, you need to add a reference to it while it's not inside a container, using g_object_ref(). If you don't want to use widget again it's usually more efficient to simply destroy it directly using Gtk.destroy since this will remove it from the container and help break any circular reference count cycles.

widget

a current child of container
